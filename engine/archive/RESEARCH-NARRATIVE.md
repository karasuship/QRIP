## 研究の出発点

**問い**: 「脳死DCAより少しでもましな積立ルールはチャートだけから見つかるか？」

前提として竹田が明示した思想:
- 劇的な改善は期待しない。小さな最適化があればいい。
- 失敗の記録も絞り込みの材料になる。
- 後付けでも、検証から一定の根拠があれば暫定的に正しいと扱う。

研究はこの前提のもとで進んだ。

---

## フェーズ1: 「待つ」戦略の可能性を検証（0047〜0059）

### 問いの立て方

最初のアプローチは「安値を待って買う」という直感を定量化すること。  
具体的に: 月初から投資を待ち、一定以上の下落があった日に買う、という戦略を評価した。

評価の枠組みとして期待値（EV）を定義した:  
```
EV = P(X日以内にY%下落) × 平均ディスカウント
   - (1-P) × 待機中のドリフト損失
```

### 発見した壁: 構造的カップリング

**核心の発見**: P（下落確率）と drift（上昇率）は全条件で正に相関している。

つまり「下落しやすい局面は上昇もしやすく、下落しにくい局面は上昇も緩やか」。  
これはEVを常に負にする構造的な壁だった。条件をどれだけ変えても、P×Dとドリフト損失がセットで動くため、待機のEVが正になる条件を見つけられなかった。

### W条件: 最初の突破口（ただし実用にならず）

vol20 < p25 AND dist200 > p75 の組合せ（W条件）だけが、P↓ AND drift↓ を同時に達成する唯一の条件だった。これはカップリングを破る最初の事例。

しかし 0059 のシミュレーションで明確になった: **W条件月は全体の10〜13%にすぎず、この月にだけ待機しても DCA と実質同等**（TEST +0.0%）。

**フェーズ1の結論**: 「待つ」という発想自体が間違っていた可能性が高い。機会損失（待機中の上昇を逃す）は思ったより深刻で、条件を絞っても補いきれない。

## フェーズ2: 発想の転換——「待つ」から「落ちた後に買う」へ（0060〜0061）

### 転換点の発見

0060 でα/β/γ（月末効果・曜日効果・待機ウィンドウ）を検証していたとき、δ（急落後リバウンド）が突出した数値を出した。

**δ: 1日-2%以上の急落の後、21〜63日後のリターンが体系的に高い**

```
21日後: TRAIN +1.12%  TEST +1.73%
63日後: TRAIN +0.59%  TEST +4.69%
```

これがフェーズ1との根本的な違いだと気づいた:

| | WAIT戦略 | δ戦略 |
|---|---|---|
| 落ちる「前」に待つ | YES | NO |
| 機会損失が発生するか | する | **しない**（落ちてからなので） |

落ちてから買うなら、待機コストがゼロ。EVの計算式そのものが変わる。

### 実装して確認（0061）

δをDCAに組み合わせた: 「毎月DCA + -2%急落日に0.5単位追加（月1回上限）」

**結果**: FULL +1.46%、TRAIN +0.41%、TEST +0.25% ← 全期間一貫してプラス。**DCAを初めて安定して上回った最初の戦略。**

ボーナス投入のROI単独でもDCA通常投入より1.75%高い（TEST）。これはシグナルが機能していることの直接証拠。

D1（-1%）は TEST -0.45% と逆効果。「どんな下落でも追加」ではなく、-2%以上という閾値に意味があった。

## フェーズ3: バリエーション探索と ATH-tier の発見（0062〜0063）

### まず「上限撤廃」を試みて失敗

D2 が有効なら上限なしでもっと買えばいいのでは？  
→ D2 uncapped: TRAIN -7.56%（大失敗）

理由が重要: 2001〜2002年（ドットコム）、2007〜2009年（金融危機）に unlimited buy がトリガーされ続け、価格が下がり続ける市場で無限に買い増した。**月次上限は「リスク制御」として必須**と確定。

### H2b の数字が示した手がかり

新仮説群（H2〜H8）のなかで、H2b（ATH から -10% 以下の時にボーナス買い）のボーナス単独ROIが突出した:

```
H2b bonus_ROI TEST = 1.1684 (vs DCA = +16.84%)
```

この数字を見た瞬間に気づいた疑問:  
**「なぜ ATH からの乖離が品質フィルタとして機能するのか？」**

ただし H2b は TRAIN -0.29%（単体では不安定）。何かが足りない。

### ATH-tier と D2 の組み合わせ

H2b の弱点は「ATH-10%以下にいるだけで買う→下落中の市場に際限なく入る」点。  
ここにD2（その日が-2%以上の下落）を同日要求として追加したのが ι2 の核心:

```
ι2: ATH -10%以下 AND 当日-2%以上
    ATH -10%〜-15%: 0.5単位
    ATH -15%〜-20%: 1.0単位
    ATH -20%以下:   1.5単位
```

**結果（ι2 cap=1）**: TRAIN +2.56%、TEST +3.67%。どちらも正。

さらに: ι2 uncapped（上限なし）はTRAIN +7.85%——D2 uncapped（TRAIN -7.56%）と天と地の差。  
ATH フィルタが「序盤の下落を避け、本物の安値圏でだけ買う」という機能を果たしていた。

**この理解がフェーズ3の核心**: ATH 乖離は価格の割安度を測る品質フィルタとして機能する。

## フェーズ4: パラメータ最適化（0064〜0066）

### cap のスイープ

cap=1〜12 を全スプリットで検証した結果:

| cap | TRAIN | TEST |
|---|---|---|
| 1 | +2.56% | +3.67% |
| 6 | +7.14% | +10.06% |
| 9 | +7.85% | +10.09% |
| 12 | +7.71% | 10.04%（プラトー） |

**cap=9 以降はほぼプラトー**。自然上限: 一ヶ月に ATH-10% AND -2% の条件を満たす日は現実には10回程度が最大。

cap を増やしても品質が下がらない理由が明確になった: フィルタが十分に選択的なので、cap 内でのトリガーはすべて「本物の局面」。

### 閾値チューニング（λ）

-10/-15/-20% を -12/-17/-22% に厳格化（λ2）すると:
- より深い調整まで待つ → 一回あたりの品質がさらに高い
- TRAIN で若干改善、TEST で同等

## フェーズ5: vol 適応サイズ（0067〜0068）

### vol 水準別の前向きリターン（υ）

ι2 トリガー日を vol 水準で分けると:

```
vol > 0.25（極度の恐怖）: 21日後 +4.94%（n=54）
vol 0.135-0.25（高め）:   21日後 +1.07%（n=25）
```

4.6倍の差。vol>0.25 は「恐怖が最大のとき」を測っており、そのタイミングが最も有効だった。

この発見から: **vol>0.25 のときにボーナスサイズを倍率で拡大**（φ）

```
φ2: vol > 0.25 ならサイズ × 2.0
TRAIN: +10.56%  TEST: +14.98%
```

また翌日買い vs 即日買いを比較すると即日が最良。「確認のために1日待つ」は損。

### 月間ドルキャップの限界

vol × 2.0 で無制限に投入すると（dollar cap=999）TRAIN は -0.53%。  
trigger count cap=6 で止めれば常に正。

**整理**: trigger 数の上限は有効。ドル無制限は NG。

## フェーズ6: Monte Carlo 統計検証（0069）

### 「偶然ではないか」の確認

80+ の戦略を試した後に「これが有効」と言っても多重比較の問題がある。  
Monte Carlo でそれを正面から検証した。

```
手法: ι2 のボーナス投入日を同期間内のランダム日に置換 × 5000回
結果: ι2 の成績を一度も超えない（100% 超過）
p値: 0.0000  Z-score: 5.73（TEST）
```

統計的には「ランダムでは説明できない」が確定した。ただし:

- 多重比較の問題は完全には消えない（最良戦略を選んだ上での確認）
- TEST 期間（2011-2026）の 75 トリガーだけで Z=5.73 は強い証拠
- 「仕組みが理解できている（ATH 品質フィルタ）」という理論的裏付けが補完

## 研究の総括

### 答えが出た問い

| 問い | 答え |
|---|---|
| WAIT 戦略で DCA を超えられるか | NO（構造的カップリングで常に EV 負） |
| どんな条件でも WAIT は無意味か | W条件は DCA と同等（プラスではない） |
| 「落ちた後に買う」は有効か | YES（δ、D2 cap=1 で確認） |
| 月次上限は必要か | YES（uncapped は TRAIN で-7%台） |
| ATH 乖離は意味を持つか | YES（品質フィルタとして機能） |
| vol は追加情報になるか | YES（vol>0.25 で効果が 4 倍） |
| シグナルはノイズか | NO（Z=5.73、p<0.0001） |

### 発見の体系

```
DCA（基準）
 └─ +0.25% ─ D2 cap=1          ← 最小の改善、原理の確認
     └─ +3.67% ─ ι2 cap=1      ← ATH フィルタで品質向上
         └─ +10.06% ─ ι2 cap6  ← cap を最適化
             └─ +14.98% ─ φ2   ← vol 水準でさらに差別化
```

### 残った問い

- Kelly 基準から見た理論的最適ベットサイズはいくつか
- 1929 型シナリオ（ATH-80%）ではどう挙動するか
- Monte Carlo での多重比較を厳密に補正すると p 値はどう変わるか
- 実際の口座で実装するときの運用ルールはどう書くか
