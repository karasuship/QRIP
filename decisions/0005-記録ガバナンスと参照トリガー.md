# 0005: 記録ガバナンスと参照トリガーの制定
- 状態: 有効
- 日付: 2026-06-20
- 起案: A・B の議論（自律提案の是非／参照の原理）→ Claude 整理 → A・B 承認
- 種別: 設計
- 確認状態: 確認済み

## 背景（経緯）

冷アーカイブを作った上で、2つの問いが出た。
1. **ガバナンス**（A・B）：核を記録するとき、AIが自律的に判断して書き込んでいいか。Aは自律提案に前向き、Bは「提案まではいいが、勝手な埋め込みは不可」。
2. **参照の原理**（B）：冷アーカイブは毎回読まれないのに、どういう基準で Claude は参照する/しないを決めるのか。Aの像は「人間が迷子のとき立ち返る」。Bは「話題（株など）を検知したら一旦見る自動参照」も要るのでは、と提起（＝手順書【F】の書庫＋重複検知の参照側）。
3. **記録の濃さ**：Aは「思考プロセスが辿れる圧縮」、Bは「会話を丸ごと」。

## 決定

- **ガバナンス**：Claude は核の追加に気づいたら**自律的に提案する。が、勝手に書き込まない**。提案→A・Bが承認→記録。最終採否は人間。PRレビューが並行作業の最終関門。
- **参照トリガー**（熱の層 CLAUDE.md・メモリ・地図に明記）：
  - 思考レベル＝方向/正当化/採否/マネタイズ/法務/憲法・核の追加変更/矛盾チェック/人間が聞いたとき、に開く。
  - 領域レベル＝特定話題の「採否・検証・書き足し」作業前に対応書庫を1回見る（ルーティング表。検証書庫は最初の発見時に新設＝枠は今、中身は後）。
  - 機械的作業・雑談では開かない（過剰回避）。
- **記録の濃さ**：階層ごとに変える。憲法＝結論／decisions・冷アーカイブ＝思考圧縮（経路を残す、ただの要約にしない）／生ログ＝控え（Claude Code のセッション記録に依る＋節目は能動退避）。

## 理由

- **自律「提案」だが自律「記録」はしない**：機械は発見者でなく処刑人／発見も審判も人間が関門、という思想と一致。気づきはAIが速い、採否は人間が持つ、の役割分担。
- **参照は「自動でも人間頼みでもなく、トリガーで」**：完全自動は過剰でノイズ、「人間が聞いたときだけ」はドリフトの穴（人間はいつ聞くべきか分からない）。中間＝明示トリガーを熱の層に置く。冷ストレージは熱のポインタの良し悪しだけ届く、という Claude の仕組みの正直な開示に基づく。100%自動を約束できないからこそ、人間が聞ける/承認する多重防御を残す。
- **濃さは源と地図の関係**：生ログ（B）＝原典・territory、思考圧縮（A）＝そこから引いた地図。片方だけにするのが唯一の誤り。両方を別の役割で持つ。
- **検証書庫を今は作らない**：発見がゼロの空箱を建てるのはオーバーエンジニアリング。枠と道（ルーティング規則）は今、中身は最初の法則が出た時。

## 廃止したもの
なし（新規制定）。

## 補足（打ち止め宣言）
ここまでで記録・ガバナンスの足場づくりは一旦完了。順番の掟「中身が先・器が後」に従い、次は実体（Step0検証 or 行動支援MVP）に移る。
