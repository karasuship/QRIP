---

## 決定の背景

QRIPは「いつ買うか（タイミング）」を30年統計で示してきた。
次の柱として「何を買うか（銘柄選別）」を定量的に提供する。

競合（Yahoo Finance・SBI・kabutan等）にも同様のスクリーナーはあるが、
**あることが前提**であり、ないことが逆差別化になる。
差別化は「不動産含み益の坪単価計算」「シグナルとの統合」でやる。

---

## 機能概要

ユーザーが条件を選択し、全上場銘柄（東証全市場）の中からヒットする銘柄を一覧表示する。

---

## スクリーニング条件（ユーザー選択式）

### 資産バリュー軸
| 指標 | 推奨デフォルト | 説明 |
|------|----------------|------|
| PBR | ≤ 0.5倍 | 簿価の半値以下。市場に忘れられている |
| 自己資本比率 | ≥ 60% | 借金が少なく財務健全 |
| 配当利回り | ≥ 2% | 待ちながら取れる |
| 設立からの年数 | ≥ 30年 | 老舗。含み資産が蓄積されやすい |

### 収益バリュー軸
| 指標 | 推奨デフォルト | 説明 |
|------|----------------|------|
| PER | ≤ 10〜15倍 | 利益に対して安い |
| 営業利益率 | ≥ X% | 本業できちんと稼いでいる |

### フィルター
| 条件 | 選択肢 |
|------|--------|
| 市場区分 | 全て / プライム / スタンダード / グロース |
| 時価総額 | 下限・上限（任意） |

---

## 差別化レイヤー（Phase 2以降）

### 不動産含み益・坪単価（最重要差別化）
- 有報（EDINET XBRL）から土地の帳簿価額（取得原価）を取得
- 国交省の地価公示APIで現在の公示地価と突き合わせ
- 「帳簿上の土地価値 vs 現在価値」の差額を可視化
- 坪単価で比較できる形に整形
- これをやっているウェブサービスはほぼ存在しない

### シグナルとの統合
- スクリーナーで銘柄を絞り込む
- SP500シグナルが「買い場」を発動したタイミングで一括購入
- 「銘柄選別（資産バリュー）× タイミング（phi2）」の組み合わせ

## データソース

| データ | API | 備考 |
|--------|-----|------|
| 財務データ（PBR/PER/自己資本比率/利回り） | J-Quants API（JPX公式・無料） | アカウント取得要 |
| 株価・市場区分・時価総額 | J-Quants API | 同上 |
| 会社設立日 | J-Quants 企業マスタ | 同上 |
| 不動産帳簿価額 | EDINET XBRL API | Phase 2 |
| 地価公示 | 国土交通省 不動産情報ライブラリAPI | Phase 2 |

3. Supabaseから条件フィルタリングしてリアルタイム表示（サーバーサイド）
