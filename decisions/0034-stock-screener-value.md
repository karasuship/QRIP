**日付**: 2026-06-26  
**状態**: 構想中（未実装）  
**担当**: 竹田  
**前提条件**: J-Quants APIアカウント取得後に実装着手

---

## 決定の背景

QRIPは「いつ買うか（タイミング）」を30年統計で示してきた。
次の柱として「何を買うか（銘柄選別）」を定量的に提供する。

競合（Yahoo Finance・SBI・kabutan等）にも同様のスクリーナーはあるが、
**あることが前提**であり、ないことが逆差別化になる。
差別化は「不動産含み益の坪単価計算」「シグナルとの統合」でやる。

---

## 機能概要

ユーザーが条件を選択し、全上場銘柄（東証全市場）の中からヒットする銘柄を一覧表示する。

---

## スクリーニング条件（ユーザー選択式）

### 資産バリュー軸
| 指標 | 推奨デフォルト | 説明 |
|------|----------------|------|
| PBR | ≤ 0.5倍 | 簿価の半値以下。市場に忘れられている |
| 自己資本比率 | ≥ 60% | 借金が少なく財務健全 |
| 配当利回り | ≥ 2% | 待ちながら取れる |
| 設立からの年数 | ≥ 30年 | 老舗。含み資産が蓄積されやすい |

### 収益バリュー軸
| 指標 | 推奨デフォルト | 説明 |
|------|----------------|------|
| PER | ≤ 12倍（10〜15倍で選択可） | 利益に対して安い |
| 営業利益率 | ≥ X%（業種平均比も将来的に） | 本業できちんと稼いでいる |
| ROE | ≥ 5〜8% | PBR低 × ROEそこそこ = 本物のバリュー株 |
| ROA | ≥ X% | 総資産に対してどれだけ稼いでいるか。ROEより誤魔化しにくい |

### 規模・流動性
| 指標 | 選択肢 |
|------|--------|
| 時価総額 | 下限・上限（任意）|
| 市場区分 | 全て / プライム / スタンダード / グロース |

### 成長性（品質フラグ付き）
| 指標 | 考え方 |
|------|--------|
| 売上成長率（前年比） | ≥ X% で「成長している」フラグ |
| 売上成長率の上限警戒 | **20%超は無理な拡大の可能性**。急成長は過剰債務・在庫積み上げ・出店過多のリスク。フラグ表示 |
| 3期連続成長 | 安定成長か一時的な跳ねかを判別 |

### 業種別指標（Phase 2以降）
| 業種 | 指標 | 内容 |
|------|------|------|
| 小売業 | 出店余地 | 既存店舗数 vs 国内余地。決算短信の「店舗数推移」から推計 |
| 不動産・製造業 | 不動産含み益 | 下記「差別化レイヤー」参照 |

## 差別化レイヤー（Phase 2以降）

### 不動産含み益・坪単価（最重要差別化）
- EDINET XBRLから土地の帳簿価額（取得原価）を取得
- 国交省の地価公示APIで現在の公示地価と突き合わせ
- 「帳簿上の土地価値 vs 現在価値」の差額を可視化
- 坪単価で比較できる形に整形
- これをやっているウェブサービスはほぼ存在しない

### シグナルとの統合
- スクリーナーで銘柄を絞り込む
- SP500シグナルが「買い場」を発動したタイミングで一括購入
- 「銘柄選別（資産バリュー）× タイミング（phi2）」の組み合わせ

## 売上急成長フラグの設計思想

売上成長率が高いことは必ずしも良くない。特に以下のリスクがある：

- **20%超**: 店舗過剰出店・設備投資過多・在庫積み上げのサイン。キャッシュフローが悪化していることが多い
- **急成長 × 自己資本比率低下**: 借入で成長を買っているパターン。危険
- **急成長 × 営業利益率低下**: スケールしているが稼げていない

表示方針：成長率に応じて「安定成長 / 急成長（要注意） / 縮小中」のラベルをつける。数字を出すだけでなく解釈を渡す。

## 実装方針（Phase 1）

1. J-Quants APIでSupabaseに全銘柄の財務データを夜間バッチ蓄積
2. `/screener` ページにインタラクティブな条件UI
3. Supabaseから条件フィルタリングして表示
4. 成長率フラグは Supabase に前処理して保存（3期分の売上を計算してカラム化）
