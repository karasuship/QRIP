# 0006: 検証エンジンの方針と検証データの信頼設計
- 状態: 有効
- 日付: 2026-06-20
- 起案: A・B の議論 → Claude 整理 → A・B 承認
- 種別: 設計
- 確認状態: 確認済み

## 背景（経緯）

「手作業で仮説を1個ずつ打つ地獄をやめ、自律的に探索したい」(A)。だが解釈は無限にあり手当たり次第は過学習に直行する(B)。一方「まともな因果が通る仮説」は把握できる範囲(A)。さらに「後で立ち返った時にデータを信じていいか」を決める信頼設計が要る(A)。

ゴールは積立投資家を上回ること。最初の標的は「長期ではノイズだが短期では割安な下落局面（例：-10%で買えば積立てない人より安く・多く仕込める）」を見抜くこと（＝Step0／柱B）。

軸の出発点を巡り、A「チャートが集合知の結果だから第一軸」、B「普段は綺麗なチャートを描く恒常項（石油等）が、危機時に失われると大ノイズが走る」と議論。両者は対立でなく合流した（下記）。

## 決定

1. **機械は処刑人、発見・意味づけは人間。** 因果の関門（地雷①）が「無限の解釈」を「有限の仮説」に変換する。自律性は "実行（物量）" に置き、"意味の発見" には置かない。
2. **原理不問 ≠ 検証不問。** 説明できなくても偶然でも候補にするが、OOS生存は必須。
3. **多重検定を必ず扱う。** 試行回数を記録し、回数に応じて有意の基準を上げる。全保存（○）と素朴な有意性での採用（✗）は別物。
4. **信頼設計4原則**：(a) 原理不問・生存で採る (b) 何も証明済みにならない＝暫定生存 (c) 常時の再検証と取捨選択、崩れたら降格・引退（書庫から消さない） (d) 全結果に素性ラベル（どう出した／どの期間／OOS／試行回数の文脈／現在の地位）。
5. **保存は一律フラットに全保存＋素性ラベル。** 優先順位は "保存" でなく "派生ビュー（ラベルへの問い合わせ）"。事実（不変の記録）と信頼（導出される解釈）を混ぜない。
6. **検証順**：単軸OOS → 生存者だけ交互作用 → 横断比較は書庫への問い合わせ。
7. **今ロックする項目**：①結果レコードのスキーマ ②地位の語彙と遷移規則 ③OOS分割の固定方針＋試行回数の記録 ④保存先 `engine/`（最初はファイル、DB不要）。
   - スキーマ案：`指紋 / 軸名 / パラメータ / 因果の理由 / 対象市場 / 期間(train・test) / 結果(n・平均・勝率・最悪・分布) / OOS判定(前半○後半○×) / 試行回数の文脈 / 地位(候補/生存/過学習疑い/引退) / 検証日`
   - 地位の遷移：候補→生存（OOS通過）／生存→過学習疑い（後半で崩れた）／→引退（再検証で落ちた）。
8. **第一軸はチャート（下落幅→将来リターン分布）。** 世界情勢・資源・地政学は別軸でなく**レジーム（状態）ラベル**として危機時に条件付きで層を成す。チャート＝攻め（エッジ）、レジーム＝守り（大負け回避／分散低減）。
   - **軸の生成法**：ノイズ＝「前提とされ織り込まれた恒常項」の一時的逸脱への過剰反応（[[二次的な核]]参照）。**恒常項（石油・金利・信用・サプライチェーン・通貨・電力…）は有限で列挙できる**ので、軸は「全事象」でなく「主要な恒常項とその逸脱」を列挙して空間を有限化する。逸脱が一時的なら過剰恐怖（狙う）、恒久的なら正当下落（避ける）。

## 理由

- **1〜3**：自動探索は偽相関を量産する（地雷⑤）／「52が本物か見抜く方が100倍難しい」／チャートにフィットさせる＝当てるへの逆戻り（第1段で放棄）を避ける。試行回数を増やすほど偶然の当たりも増える（多重検定）。
- **4〜5**：データの価値は数字でなく**素性**で決まる。協働スタイル「確認済／未確認／要調査」を検証書庫に適用。decisions と同じ「消さない・地位ラベルで廃止」の作法で一貫。
- **8（二本の矢印）**：A＝帰納（チャート事例→法則）、B＝演繹（恒常項の原理→個別）。両者は合流し、**演繹（人間）が意味ある軸を立てて空間を有限化し、帰納（機械）が歴史で生存を裁く**＝そのまま「発見は人間・審判は機械」。生の因果を常時追うとサンプル死（第I部6）なのでレジーム圧縮し危機時のみ起動。サンプルが小さいので危機要因は勝率を上げず、効くのは大負け回避（分散低減）。
